一、导入SDK

Gradle 编译环境(Android Studio)

1.将repo.zip解压至任意路径下
2.在Project级别的build.gradle文件中添加 wmdaplugin 依赖： 
buildscript {
    repositories {
        maven{
       	    // 由于代码保护问题，故没将pulgin上传到maven中⼼仓库。
     	    // 所以需要添加本地 maven repo
     	    // 其中'/xxx/xxx/xxx/repo'为repo.zip解压的路径
     	    url uri('/xxx/xxx/xxx/repo')
        }
        jcenter()
    }
    dependencies {
        classpath 'com.android.tools.build:gradle:2.3.0'
        // 添加wmdapulgin依赖
        classpath 'com.wuba.wmdaplugin:wmdaplugin:1.0.0'
    }
}
3.将wmda.aar移动⾄module下的libs⽬录内，例如 XXProject/app/libs/
4.在Module级别的build.gradle文件中添加 wmdaplugin 和 wmda 的依赖： 
apply plugin: 'com.android.application'
// 使⽤wmdaplugin
apply plugin: 'com.wuba.wmdaplugin'
···
repositories {
    flatDir {
    dirs 'libs'
    }
}
dependencies {
    ···
    // 添加wmda的依赖
    compile(name:'wmda', ext:'aar')
    ···
}

二、初始化

请在项目 Application 的 onCreate 方法中对WMDA进行初始化： 
public class MyCustomApplication extends Application {
    @Override
    public void onCreate() {
        super.onCreate();
        // 应⽤ID，具体指向服务端索取
        String appid = "3";
        // 应⽤ID对应key，具体指向服务端索取
        String appkey = "12345678";
        // 事件存储加密秘钥，⽤于对本地事件的加密存储
        // 请勿频繁更改，秘钥更改后，之前加密的内容将⽆法解密上报
        // 如果encryptKey不设或者值为空字符串，则默认不进⾏加密
        String encryptKey = "";
        // 渠道ID
        String channelID = "wandoujia";
        // 设置debug开关，release版本请将其设为false
        boolean isDebug = true;
        WMDA.init(this,appid,appkey,encryptKey,channelID, isDebug);
    }
}
注意事项：
请确保将代码添加在 Application 的 onCreate 方法中，添加到其他方法中可能导致数据不准确。

三、必要权限

使⽤WMDA需要以下权限： 
 <!-- WMDA上报需要⽹络权限 -->

五、圈选添加URL Scheme

将下面的Intent-filter添加到AndroidManifest.xml中的LAUNCHER Activity下，以便我们唤醒您的App，进入圈 选。 
<!-- WMDA 启动圈选接入 -->
<intent-filter>
    <data android:scheme="wmda.58"/>
    <action android:name="android.intent.action.VIEW"/>
    <category android:name="android.intent.category.DEFAULT"/>
    <category android:name="android.intent.category.BROWSABLE"/>
</intent-filter>
<!-- WMDA 启动圈选接⼝ -->
注意事项： 请将整个intent-filter添加到LAUNCHER Activity中,并确保其中只有一个data字段。

六、重要配置项

设置界面元素ID

当您的App界面改版时，可能会导致方法准确地统计已经圈选的元素。因此，对于应用中的主要流程涉及到的界面元素，建议您对它 们设置固定的唯一ID，以保证数据的一致性。
WMDA.setViewID(View view, String viewID) 其中，第一个参数是要设置的View，第二个是给这个View的ID。

采集GPS数据

App获取GPS坐标后，可以调用如下接口进行设置经纬度。 
WMDA.setLocation(String latitude, String longitude)
注意事项： Android端WMDA SDK不会自动获取GPS数据，如果要采集GPS数据，请在获取到GPS信息后调用setLocation()接口设置。

设置采集Area

特殊接口，用于一些业务线的特殊场景使用，如不需要，可不使用该接口。 
WMDA.setArea(String area)

设置GPSArea

特殊接口，用于一些业务线的特殊场景使用，如不需要，可不使用该接口。 
WMDA.setGPSArea(String gps_area)

采集广告Banner数据

对于横向滚动的Banner广告，如果该banner是通过 ViewPager、AdapterView 或者 RecyclerView 实现的，请在Banner创建时（包括动态创建）调用下面的接口来采集数据。 
WMDA.traceBanner(View banner)

七、自定义维度配置项

为了能够让数据分析变得更加的灵活，我们提供了自定义维度的接口，使用者可以对各种对象设置属性。这些属性在作图时，将表现为维度。

用户属性

用户属性只能用来表示登录用户本身的属性，至少包括用户ID。设置UserID接口。 
WMDA.setUserID("userid");
根据需求，业务线还可以指定用户的各种属性，用户属性被称为US字段，最多支持 9 个，从 US1到 US9 。自然属性，比如性别、出生年月等。 账户属性，比如等级、类型标签等。 行为特征，比如是否有过购买记录之类。 设置用户属性接口如下： 
WMDA.setUS1("US1的key","US1的value");
...
WMDA.setUS8("US8的key","US8的value");
WMDA.setUS9("US9的key","US9的value");
注意事项：
1. US字段不能是和用户没有直接关系的属性，比如不能是订单，ID商品ID等。
2. 对于未登录用户，不要设置任何US字段。
3. 如果没有用到所有的US字段，剩下的可以不设置。
4. US字段是有序的，比如可以只设置US1和US3
5. 同一个US字段，必须保持在各个平台意义相同。
设置用户信息时机
基本原则：当App使用者的登录状态改变时调用 WMDA.setUserID 接口设置用户ID。
用户手动登录：
1. 如果有多个登录入口，在每一个入口登录成功后，都需要调用 WMDA.setUserID 来设置用户的唯一标识。
2. 如果有第三方登录，成功登录后需要调用WMDA.setUserID接口。

自动登录：App启动时，自动登录或者用户默认是登录状态，也需要调用 WMDA.setUserID 接口。
注册：有的App注册成功后，默认登录，这种情况下也需要调用 WMDA.setUserID 接口。
退出：退出登录时，只需要 WMDA.setUserID(null) ，UerID置空即可。 设置US字段属性信息同设置用户ID时机和方式一致，这里不再重复描述。

页面属性

页面属性用来表示当前页面的属性，至少包括页面ID。设置PageID接口。 下面我们以Activity为例进行说明。如果您设置的的对象是Fragment，将下面的Activity替换为Fragment即可。 
WMDA.setPageID(this,"id_MainActivity");
根据需求，业务线还可以指定页面的各种属性，页面属性被称为PS字段，最多⽀持 9 个，从 PS1到 PS9 。 您可以将商品的类别，尺码、颜色、生产厂商等信息设置为PS字段，从而更加灵活的分析用户行为。 设置页面属性接口如下： 
WMDA.setPS1("PS1的key","PS1的value");
...
WMDA.setPS8("PS8的key","PS8的value");
WMDA.setPS9("PS9的key","PS9的value");
比如定义产品详情页，并上传4个页面属性
分别是：
PS1：产品ID：1234567
PS2：产品名称：T-shirt
PS3：产品尺码：M
PS4：产品颜色：蓝色 
protected void onCreate(Bundle savedInstanceState) {
    super.onCreate(savedInstanceState);
    setContentView(R.layout.activity_main);
    WMDA.setPS1(this, "1234567");
    WMDA.setPS2(this, "运动T-shirt");
    WMDA.setPS3(this, "M");
    WMDA.setPS4(this, "蓝⾊");
}
注意事项：
1. 页面属性接口必须在Activity的onCreate方法中调用。
2. 如果没有用到所有的PS字段，剩下的可以不设置。
3. PS字段是有序的，比如可以只设置PS1和PS3。
4. 除了页面本身外，在商品详情页上面所有的界面元素的行为，包括浏览和点击等，都会和这些页面属性建立关联。

设置CateID

设置CateID是特殊接口，各个业务线可以根据需要使用，如果App没有CateID概念，可以不用关心该接口。SDK提供对View和页面(Activity和Fragment)设置CateID接口。
1.对View设置CateID： 
WMDA.setCateID(View view, String cateID)
2.对页面设置CateID，以Activity为例，Fragment同理： 
WMDA.setCateID(Activity activity, String cateID)

统计自定义事件

为满足App不同场景事件统计需求，可以使用自定义事件灵活进行统计。 
WMDA.trackEvent(String eventID, String cateID, Map properties)
注意事项： 调用自定义事件接口WMDA.trackEvent，eventID不能为空，否则统计数据无效。CateID可以为空，可以根据App场景决定是否传递CateID。

八、使用须知

代码注入

wmdapugin 会对部分业务代码进行注入，请悉知。
以下是代码注入规则：
1.会对父类为 android.app.Fragment 的类进行注入。
onResume 中注入 WmdaAgent.onFragmentResume 方法
onPause 中注入 WmdaAgent.onFragmentPaused 方法
onHiddenChanged 中注入 WmdaAgent.onFragmentHiddenChanged 方法
setUserVisibleHint 中注入 WmdaAgent.setFragmentUserVisibleHint 方法
2.会对父类为 android.support.v4.app.Fragment 的类进行注入
onResume 中注入 WmdaAgent.onSupportFragmentResumed 方法
onPause 中注入 WmdaAgent.onSupportFragmentPaused 方法
onHiddenChanged 中注入 WmdaAgent.onSupportFragmentHiddenChanged 方法
setUserVisibleHint 中注入 WmdaAgent.setSupportFragmentUserVisibleHint 方法
3.会对实现了 android.view.View.OnClickListener 的类进行注入。
onClick 中注入 WmdaAgent.onViewClick 方法
4.会对实现了 android.content.DialogInterface.OnClickListener 的类进行注入。
onClick 中注入 WmdaAgent.onDialogClick 方法
5.会对实现了 android.widget.AdapterView.OnItemClickListener 的类进行注入。
onItemClick 中注入 WmdaAgent.onItemClick 方法
